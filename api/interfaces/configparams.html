# Engine

# chooseAddressMappingPolicy

• chooseAddressMappingPolicy: ChooseAddressMapping

Defined in src/Config.ts:66

Determines which address mapping policy will be used. Built in implementations:

  • DenseSparseChooseBasedOnThreshold - will choose address mapping for each sheet separately based on fill ratio.
  • AlwaysDense - will use DenseStrategy for all sheets.
  • AlwaysSparse - will use SparseStrategy for all sheets.

default AlwaysDense

# gpuMode

• gpuMode: GPUMode

Defined in src/Config.ts:151

Allows to set GPU or CPU for use in matrix calculations. When set to 'gpu' it will try to use GPU for matrix calculations. Setting it to 'cpu' will force CPU usage. Other values should be used for debugging purposes only. More info can be found in GPU.js documentation.

default 'gpu'

# matrixDetection

• matrixDetection: boolean

Defined in src/Config.ts:187

Enables numeric matrix detection feature when set to 'true'. During build phase each rectangular area of numbers will be treated as one matrix vertex in order to optimize further calculations. Some CRUD operations may break numeric matrices into individual vertices if needed.

default true

default 100

# useColumnIndex

• useColumnIndex: boolean

Defined in src/Config.ts:271

Switches column search strategy from binary search to column index. Used by VLOOKUP and MATCH functions. Using column index may improve time efficiency but it will increase memory usage. In some scenarios column index may fall back to binary search despite this flag.

default false

# Date and Time

# dateFormats

• dateFormats: string[]

Defined in src/Config.ts:78

A list of date formats that are supported by date parsing functions.

The separator is ignored and it can be any of '-' (dash), ' ' (empty space), '/' (slash).

Any order of YY, MM, DD is accepted as a date, and YY can be replaced with YYYY.

default ['MM/DD/YYYY', 'MM/DD/YY']

# leapYear1900

• leapYear1900: boolean

Defined in src/Config.ts:169

Preserves an option for setting 1900 as a leap year. 1900 was not a leap year, but in Lotus 1-2-3 it was faulty interpreted as a leap year. Set to true for compatibility with Lotus 1-2-3 and Excel. See nullDate for complete solution.

default false

# nullDate

• nullDate: SimpleDate

Defined in src/Config.ts:298

Allows to set a specific date from which the number of days will be counted. Dates are represented internally as a number of days that passed since this nullDate.

default {year: 1899, month: 12, day: 30}

# precisionEpsilon

• precisionEpsilon: number

Defined in src/Config.ts:226

Controls how far two numerical values need to be from each other to be treated as non-equal. a and b are equal if they are of the same sign and: abs(a) <= (1+precisionEpsilon) * abs(b) and abs(b) <= (1+precisionEpsilon) * abs(a). It also controls snap-to-zero behavior for additions/subtractions: for c=a+b or c=a-b, if abs(c) <= precisionEpsilon * abs(a), then c is set to 0

default 1e-13

# precisionRounding

• precisionRounding: number

Defined in src/Config.ts:235

Sets how precise the calculation should be. Numerical outputs are rounded to precisionRounding many digits after the decimal.

default 14

# smartRounding

• smartRounding: boolean

Defined in src/Config.ts:260

Sets the rounding. If false, no rounding happens, and numbers are equal if and only if they are truly identical value (see: precisionEpsilon).

default true
